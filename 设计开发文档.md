# 墨核 (Mohe) 设计开发文档

版本：v0.4.0
日期：2026-06-18

---

# 一、项目概述

墨核（Mohe）是一款基于 Tauri 2 构建的本地优先（Local-First）AI 驱动 Markdown 知识管理桌面应用。

核心目标：

- 提供高质量 Markdown 编辑体验
- 深度集成大模型能力（流式对话 + 工具调用）
- 构建本地 RAG 知识检索系统
- 提供长期记忆机制
- 确保数据本地化与安全存储

---

# 二、总体架构

## 2.1 架构分层

系统采用分层架构：

1. 表现层（React UI）
2. 状态层（Zustand Store）
3. 业务服务层（AI / RAG / Agent / Memory）
4. 数据访问层（SQLite）
5. 桌面能力层（Tauri Rust 后端）


## 2.2 技术栈

### 桌面框架
- Tauri 2
- Rust

### 前端
- React 18
- TypeScript 5.7
- Vite 6

### 编辑器
- CodeMirror 6
- Yjs（协同编辑预留）

### 状态管理
- Zustand 5
- debounce 增量持久化

### 数据库
- SQLite（Tauri SQL 插件）
- sqlite-vss 向量扩展

### AI 能力
- OpenAI 兼容 API
- Ollama 本地模型（Embedding + 对话）
- 流式响应（SSE / Fetch Stream）

### 样式系统
- Tailwind CSS 3.4
- Animal Island UI

# 三、功能模块设计

## 3.1 编辑器模块

功能：

- Markdown 编辑
- 实时预览
- 分屏模式
- Diff 模式
- 正则搜索替换
- 多标签页管理
- 大文档虚拟滚动

技术实现：

- CodeMirror 6 扩展机制
- remark / rehype 生态
- KaTeX 数学公式渲染
- Mermaid 图表支持
- Yjs CRDT（协作预留）


## 3.2 AI 助手模块

功能：

- 流式对话
- 多模型支持
- 上下文附加
- 系统提示词模板

架构流程：

1. 构建上下文
2. 发送请求至 AI Provider
3. 流式接收响应
4. Markdown 渲染
5. 可选工具调用解析


## 3.3 Agent 工具调用系统

核心流程：

1. 意图识别（Intent Detection）
2. 工具选择（Tool Selector）
3. LLM 生成工具调用 JSON
4. 解析工具调用
5. 并行执行工具
6. 汇总结果返回模型


## 3.4 RAG 检索增强生成

流程：

1. 文档分块（标题层级 + 固定长度混合分块）
2. 向量生成（Embedding）
   - 外部 API（OpenAI compatible）
   - 本地模型（Ollama nomic-embed-text，零费用）
3. 存储至 SQLite + sqlite-vss 向量表
4. 查询时计算余弦相似度
5. Top-K 召回
6. 上下文注入

## 3.5 长期记忆系统

- 陈述性记忆（知识点）与程序性记忆（操作习惯）分类存储
- 记忆衰减机制，避免知识库无限膨胀
- 用户手动保存与锁定
- 语义搜索 + 记忆融合（相关记忆压缩为摘要后注入上下文）


## 3.6 Agent 工具调用系统

核心流程：

1. 意图识别（JSON Schema 驱动的 Tool Registry）
2. 工具选择
3. LLM 生成工具调用 JSON
4. 解析工具调用
5. 并行执行工具（带超时 + 熔断机制）
6. ReAct 循环（复杂任务自动迭代）
7. 汇总结果返回模型

## 3.7 数据存储设计

核心表：

- documents
- memories
- rag_chunks
- settings
- chats

安全增强：

- API Key 支持用户自定义密码加密（PBKDF2）
- Workspace 快照 / 备份机制

# 四、安全设计

1. API Key 使用 Windows DPAPI 加密存储
2. 所有文件访问限制在用户选择的 Workspace
3. Rust 层路径校验
4. 本地优先，无远程服务器
